# IT-Master — мобильный API (анализ для Android-клиента)

Бэкенд: Laravel 12, маршруты в `routes/api.php`. Стандартный префикс Laravel для API — **`/api`**.

## Базовый URL

```
{BASE}/api/mobile/...
```

`BASE` — происхождение сайта без завершающего слэша. **Продакшен:** `https://axobeast.ru`. **Локально:** `http://10.0.2.2:8000` (эмулятор → хост).

## Условия доступа

- Учётная запись **активна** (`active = 1`).
- У роли есть право **`schedule_my`** (страница «Расписание» для студента). Иначе `403` с текстом «Нет доступа к расписанию».
- Расписание в ответе будет **пустым**, если у сотрудника не задан `group_id` (нет привязки к группе).

## Эндпоинты

### `GET /api/mobile/health`

Проверяет подключение к БД **`mysql_remote`**. На установках только с SQLite может стабильно отдавать **503** — это **не** индикатор работоспособности логина через основную БД. Для клиента опционален.

**Успех (200):** `{ "ok": true, "maintenance": false, "message": "..." }`  
**Техработы (503):** `{ "ok": false, "maintenance": true, "message": "..." }`

### `POST /api/mobile/login`

`Content-Type: application/json`

```json
{ "login": "user", "password": "secret" }
```

**Успех (200):**

```json
{
  "token": "<64 байта случайных символов>",
  "expires_in": 2592000,
  "user": {
    "id": 1,
    "login": "...",
    "fio": "...",
    "email": "...",
    "group_id": 2,
    "group_name": "ИС-21"
  }
}
```

**401:** неверный логин/пароль — `{ "message": "..." }`  
**403:** деактивирован или нет права расписания.

Токен передавать в заголовке: **`Authorization: Bearer <token>`**.

### `POST /api/mobile/logout`

Заголовок `Authorization: Bearer ...` — инвалидирует токен в кэше.

**200:** `{ "message": "Сессия завершена." }`

### `GET /api/mobile/me`

Требуется Bearer-токен.

**200:** `{ "user": { ... как в login } }`

### `GET /api/mobile/schedule`

Требуется Bearer-токен и право `schedule_my`.

Query:

- `week` (опционально) — любая дата недели; сервер берёт **понедельник** этой недели (`Carbon::startOfWeek(MONDAY)`). Формат: строка, понятная `Carbon::parse`, напр. `2026-04-21`.

**200:**

```json
{
  "week_start": "2026-04-20",
  "group": "ИС-21",
  "entries": [
    {
      "id": 1,
      "weekday": 1,
      "weekday_label": "Понедельник",
      "lesson_slot": 1,
      "start_time": "09:00:00",
      "end_time": "10:30:00",
      "subject": "Дисциплина",
      "teacher": "ФИО",
      "room": "101",
      "building": "1",
      "building_label": "Корпус 1"
    }
  ]
}
```

Порядок: `weekday`, затем `start_time`.

## CORS

Если приложение ходит напрямую с устройства (не из браузера), CORS не участвует. Для отладки через веб-просмотр может понадобиться настройка CORS на сервере.

## Рекомендации по безопасности (клиент)

- Хранить токен в **EncryptedSharedPreferences** или DataStore + шифрование.
- Использовать **HTTPS** в production.
- В будущем на сервере имеет смысл **rate limiting** на `/login` (сейчас в проекте не настроен).

## Android-клиент в этом репозитории

Каталог **`it-master-android`** — приложение на Kotlin + Jetpack Compose.

**Сборка:** откройте папку в **Android Studio** (желательно Hedgehog / Iguana или новее), дождитесь синхронизации Gradle, подключите SDK, выполните **Run** на эмуляторе или устройстве.

**Продакшен:** в приложении по умолчанию подставляется `https://axobeast.ru` (см. `DEFAULT_SERVER_URL` в `app/build.gradle.kts`).

**Эмулятор и локальный Laravel (`php artisan serve`):** замените адрес на `http://10.0.2.2:8000`.

**Пакет:** `ru.itmaster.schedule`, minSdk **26**.

### Временный обход ошибки HTTPS (только debug-сборка)

Если на сервере ещё не настроена полная цепочка сертификата, можно собрать **debug APK** с отключённой проверкой SSL (небезопасно: возможен перехват логина и токена). В **release** эта опция **всегда выключена**.

1. В корне проекта в файле **`local.properties`** (он в `.gitignore`) добавьте строку:  
   `unsafeSsl=true`
2. Соберите: `gradlew :app:assembleDebug`
3. После исправления сертификата на хостинге удалите `unsafeSsl` или поставьте `unsafeSsl=false`.

Без этой строки поведение прежнее: строгая проверка HTTPS.
